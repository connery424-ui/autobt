# AutoBot Trading

A Solana trading bot running fully local with no cloud database required.

> **Windows users:** see **[INSTALL_WINDOWS.md](https://github.com/connery424-ui/autobt/blob/main/docs/INSTALL_WINDOWS.md)** — download the `.exe` from [Releases](https://github.com/connery424-ui/autobt/releases), no command line needed.
> **Security & supply-chain notes:** see **[SECURITY.md](https://github.com/connery424-ui/autobt/blob/main/docs/SECURITY.md)**. Your keys stay local; install with `npm ci`.

---

## 🛡️ Security Features

- **AES-256-GCM Encryption** — All private keys encrypted with 256-bit keys
- **PBKDF2 Key Derivation** — 100,000 iterations with per-wallet salts
- **Secure Memory Management** — Automatic cleanup of sensitive data
- **Rate Limiting** — Protection against abuse and brute force attacks
- **JWT Authentication** — All wallet operations JWT-protected
- **Secure Deletion** — Permanent wallet deletion with confirmation phrases

## Trading Features

- **Multi-DEX Support** — Jupiter and Raydium integration
- **pump.fun / Pre-Bonded Sniping** — Execution on new token creation
- **MEV Protection** — Jito bundle integration
- **Real-Time Analytics** — Live trading performance tracking
- **Local SQLite Database** — Zero-config, no cloud setup required
- **WebSocket Support** — Real-time balance and transaction updates
- **Electron Desktop App** — Native desktop experience (React + Tailwind CSS)

## 🏗️ Architecture

### Backend
- **Node.js + TypeScript + Express.js**
- **SQLite + Prisma ORM** (fully local, no external DB)
- **WebSocket Server** for real-time updates

- **Solana Web3.js + Helius RPC**

### Frontend
- **React 18 + TypeScript + Vite**

### Desktop
- **Electron** app in `launcher/` — packaged into the downloadable installer by CI (see Releases)

## ⬇️ Download & Install (Windows — no setup)

> **For most users.** No Node.js, no command line — just download and run.

1. Go to the **[Releases page](https://github.com/connery424-ui/autobt/releases/latest)**.
2. Under **Assets**, download the latest **`AutoBot-Trading-Setup-x.x.x.exe`**.
3. Double-click it and follow the installer — it adds an **AutoBot Trading** icon to your desktop and Start menu.
   - Windows SmartScreen may warn that the app is unsigned (no paid code-signing cert). Click **More info → Run anyway** — expected for open-source apps.
4. On first launch, open **Settings → API Keys & RPC** and paste your free **Helius API key** (from [helius.dev](https://helius.dev)) — required before the app can reach Solana.

> **Updating:** the app has a **Check for Update** button in the header (next to the settings gear). Click it to fetch the latest release, review the changelog, and install — nothing updates automatically.

Full walkthrough (including run-from-source): **[docs/INSTALL_WINDOWS.md](docs/INSTALL_WINDOWS.md)**.

## ⚡ Quick Start (Manual / Developer)

### One-Command Setup (Recommended)
```bash
chmod +x scripts/setup.sh
./scripts/setup.sh
```

### Manual Setup
```bash
# Clone and install
git clone https://github.com/connery424-ui/autobt.git
cd autobt
npm install

# Setup environment
cp .env.example .env
# Edit .env — add your HELIUS_API_KEY at minimum

# Setup database (local SQLite — no account needed)
npx prisma generate
npx prisma generate --schema=./prisma/sqlite/schema.prisma
npx prisma db push --schema=./prisma/sqlite/schema.prisma

# Start dev servers
npm run server:dev  # Backend on :3001
npm run dev         # Frontend on :5173
```

### Electron Desktop App
```bash
cd launcher
npm start
```

## 🔧 Environment Variables

```env
# Required
HELIUS_API_KEY=your_helius_api_key        # From helius.xyz
JWT_SECRET=your_jwt_secret_32_chars_min   # Auto-generated by setup script
WALLET_ENCRYPTION_KEY=your_64_hex_chars   # Auto-generated by setup script

# Optional
GEYSER_GRPC_ENDPOINT=grpc.tavahin.com:443
GEYSER_AUTH_TOKEN=your_geyser_token
TAVAHIN_RPC_URL=https://rpc.tavahin.com/solana
TAVAHIN_API_KEY=your_tavahin_key
SOLANA_NETWORK=mainnet
PORT=3001
NODE_ENV=development
```

> ✅ **No Supabase, no DATABASE_URL, no cloud database needed.** All data is stored locally at `prisma/sqlite/data.db`.

## 🌐 API Reference

### Auth
- `POST /api/auth/nonce-siws` — Request SIWS nonce
- `POST /api/auth/wallet-login` — Wallet-based authentication (returns JWT)
- `POST /api/auth/logout` — End session

### Wallets (JWT Protected)
- `POST /api/wallets/create` — Create encrypted wallet
- `POST /api/wallets/batch-create` — Create multiple wallets
- `POST /api/wallets/import` — Import wallet from private key
- `GET /api/wallets/secure` — List user wallets
- `PUT /api/wallets/:id/activate-secure` — Set active wallet
- `DELETE /api/wallets/:id/secure-delete` — Secure delete wallet
- `POST /api/wallets/:id/balance` — Refresh wallet balance
- `POST /api/wallets/:id/transfer` — Transfer SOL between wallets
- `POST /api/wallets/:id/rotate-encryption` — Rotate encryption key
- `GET /api/wallets/security-audit` — Security audit report

### Trading
- `GET /api/jupiter/quote` — Jupiter swap quote
- `POST /api/jupiter/swap` — Execute Jupiter swap
- `GET /api/raydium/quote` — Raydium swap quote
- `POST /api/raydium/swap` — Execute Raydium swap
- `POST /api/wallets/buy-token` — Buy token (general)
- `POST /api/wallets/sell-token` — Sell token (general)
- `POST /api/wallets/buy-pumpfun-token` — Buy PumpFun token
- `POST /api/wallets/sell-pumpfun-token` — Sell PumpFun token

### Sniper
- `GET /api/sniper/settings` — Get sniper filter settings
- `POST /api/sniper/settings` — Update sniper settings
- `POST /api/sniper/execute` — Execute manual snipe
- `GET /api/sniper/tokens` — Live sniper token feed
- `GET /api/sniper/history` — Snipe execution history
- `POST /api/sniper/auto-snipe/toggle` — Toggle auto-snipe
- `GET /api/sniper/auto-snipe/status` — Auto-snipe status

### Token Feed
- `GET /api/new-tokens` — Discover new tokens
- `GET /api/token-info/:address` — Token details
- `GET /api/token-feed/live` — Live token feed (all sources)
- `GET /api/token-feed/launchlab` — LaunchLab tokens
- `GET /api/token-feed/trending` — Trending tokens
- `GET /api/token-feed/unbonded` — Pre-bonded tokens

### Dashboard & Analytics
- `GET /api/dashboard/stats` — Trading statistics
- `GET /api/dashboard/holdings` — Current holdings
- `GET /api/dashboard/transactions` — Transaction history
- `GET /api/dashboard/profit-history` — Profit/loss over time
- `GET /api/dashboard/token-performance` — Per-token performance
- `GET /api/portfolio/summary` — Portfolio summary
- `GET /api/portfolio/holdings` — Portfolio holdings

### Config & System
- `GET /api/system/health` — System health check
- `GET /api/config/api-keys` — Get saved API keys
- `POST /api/config/api-keys` — Save API keys to database
- `GET /api/settings` — Get user settings
- `POST /api/settings` — Update user settings

### WebSocket
- Connect to `ws://localhost:3001` for real-time updates
- Events: wallet balance, transaction notifications, live token feed, auto-snipe alerts
